# AssistChat

Платформа для мониторинга Telegram-чатов, транскрибации встреч и автоматизации задач — через веб-интерфейс, без кода.

---

## Что умеет платформа

| Блок | Что делает |
|---|---|
| **Telegram-мониторинг** | Слушает группы и каналы, пересылает нужные сообщения в бота |
| **ПРОМПТ** | Настраиваемая цепочка фильтров и AI-шагов для обработки сообщений |
| **Zoom / транскрибация** | Переводит аудиозаписи встреч в текст с разделением по спикерам |
| **QR-коды** | Генерирует QR-коды с логотипом, на выходе ZIP с PNG и PDF |
| **AI-ассистент** | Голосовой колл-центр и AI-агент для переговоров *(в разработке)* |

---

## Telegram-мониторинг

### Как работает

```
Telegram-аккаунт (Telethon) ──► MessageBus ──► ПРОМПТ ──► Telegram-бот (уведомления)
```

1. Подключаешь свой Telegram-аккаунт через API ID + API Hash.
2. Платформа от имени аккаунта слушает все входящие сообщения во всех группах и каналах.
3. Каждый активный ПРОМПТ фильтрует поток и отправляет нужное в бота.

### Что пересылается

- Текстовые сообщения
- Одиночные фото
- Альбомы (несколько фото с подписью) — приходят как полноценный альбом
- Голосовые и файлы — с пометкой типа

### Настройка Telegram-сессии

1. Получи API ID и API Hash на [my.telegram.org](https://my.telegram.org) → API development tools.
2. Создай ресурс **Telegram-сессия** в интерфейсе.
3. Введи API ID, API Hash и номер телефона.
4. Нажми **Активировать** — придёт код в Telegram, введи его.
5. Сессия сохраняется в БД и автоматически восстанавливается при рестарте.

## ПРОМПТ

Основной инструмент настройки логики. Один ПРОМПТ = один сценарий обработки сообщений.

### Источники

- **Telegram-сессия** — поток из всех групп подключённого аккаунта
- **Telegram-бот** — сообщения, которые пишут пользователи напрямую в бота

### Фильтры

| Параметр | Описание |
|---|---|
| Личные сообщения | Включить / выключить обработку личных сообщений |
| Группы | Включить / выключить обработку групп |
| Каналы | Включить / выключить обработку каналов |
| Whitelist | Обрабатывать только эти чаты / username |
| Blacklist | Игнорировать эти чаты / username |

Whitelist и Blacklist принимают: `@username`, `t.me/username`, `https://t.me/username`, числовой ID чата.

### Шаги обработки

Шаги выполняются последовательно. Каждый может остановить цепочку или передать управление следующему.

#### 1. Условие (без AI)
Проверяет сообщение по правилам:
- Ключевые слова в тексте
- Username отправителя

Действие при совпадении: **продолжить** или **остановить**.

#### 2. AI-анализ
Отправляет сообщение в AI с твоей инструкцией.

Действие по результату анализа:
- **Продолжить** — передать следующему шагу
- **Остановить** — не уведомлять
- **Уведомить владельца** — отправить AI-ответ в бота

#### 3. Уведомить владельца
Отправляет сообщение в Telegram-бот.

Режимы:
- **Напрямую** — пересылает оригинальный текст/фото как есть
- **Через AI** — AI формирует текст уведомления по твоей инструкции

### Формат уведомления

```
📌 ТЕСТИЛКА
ТЕЛЕГА 916, Baku Market, @georgia_nicks

Текст сообщения или фото...
```

### Настройка ПРОМПТ

1. Создай ресурс **ПРОМПТ**.
2. Выбери Telegram-сессию как источник.
3. Выбери Telegram-бота для уведомлений.
4. Укажи свой Telegram ID (получить: написать [@userinfobot](https://t.me/userinfobot)).
5. Настрой фильтры и добавь шаги.
6. Активируй ресурс.

## Zoom / Транскрибация

Загружаешь аудиозапись встречи — платформа возвращает текст с разделением по спикерам.

### Как работает

- Транскрибация через Deepgram (модель `nova-2`, язык: русский)
- Автоматическая диаризация — каждый спикер подписан отдельно
- Умная пунктуация и форматирование
- Результат доступен в интерфейсе и через AI-анализ

### Требования

- Ключ Deepgram API (добавляется в `.env` как `DEEPGRAM_API_KEY`)
- Аудиофайл в поддерживаемом формате (MP3, WAV, M4A и др.)

## QR-коды

Генератор QR-кодов с логотипом компании.

### Как использовать

1. Открой раздел **QR + Logo** в интерфейсе.
2. Введи текст или URL.
3. Загрузи логотип (PNG или TIFF).
4. Нажми **Создать** — скачается ZIP с PNG и PDF.

## Ресурсы (системные блоки)

Все функции платформы строятся из ресурсов. Каждый ресурс можно активировать, остановить или удалить независимо.

| Ресурс | Назначение |
|---|---|
| **API Keys** | Ключи AI-провайдеров: OpenAI, Gemini, Claude, Groq, DeepSeek, Mistral, xAI |
| **Telegram-сессия** | Подключение к Telegram-аккаунту (Telethon) |
| **Telegram-бот** | Бот для отправки уведомлений владельцу (aiogram) |
| **ПРОМПТ** | Правила фильтрации и обработки сообщений |
| **Zoom** | Транскрибация аудиозаписей встреч |

## Деплой

### Автоматически (GitHub Actions)

При каждом пуше в `master` GitHub Actions автоматически:
1. Подключается к серверу по SSH
2. Обновляет код
3. Пересобирает Docker-образы
4. Запускает миграции БД
5. Перезапускает сервисы

Необходимые секреты в GitHub: `SSH_PRIVATE_KEY`.

### Вручную

```bash
# На сервере
cd /var/www/assistchat
git pull origin master
docker compose -f docker-compose.prod.yml build
docker compose -f docker-compose.prod.yml up migrate
docker compose -f docker-compose.prod.yml up -d
```

### Переменные окружения (.env)

```env
DB_USER=...
DB_PASSWORD=...
DB_NAME=...
SECRET_KEY=...
DEEPGRAM_API_KEY=...   # для транскрибации Zoom
```
